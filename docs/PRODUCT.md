# PRD

## Product Name

PS2-style Startup Scene (Three.js)

## Goal

PlayStation 2 の起動画面（タワーシーン部分）を Three.js で **完全再現** する。

## Canon（原典）

> **`docs/assets/PS2 Startup Screen.mp4` がこのプロジェクトの唯一の正解である。**
> すべての実装判断はこの動画に基づく。ドキュメントの記述と動画が矛盾した場合、動画が正しい。
> 実装がこの動画と異なって見えるなら、実装が間違っている。

- ファイル: `docs/assets/PS2 Startup Screen.mp4`
- 尺: 12.14 秒（25 fps / 1920×1080）
- 構成:
  - `0.0s – 9.5s`: タワーシーン（本プロジェクトの再現対象）
  - `9.5s – 10.0s`: 暗転完了
  - `10.0s – 12.1s`: PlayStation 2 ロゴ（再現対象外）

### 原典の使い方

1. 実装前: 対象区間のフレームを `ffmpeg` で抽出し、目指すビジュアルを確認する
2. 実装中: 迷ったら動画のフレームを見る。推測しない
3. 実装後: viewport スクリーンショットと動画フレームを並べて比較する
4. 差異があれば: 動画に合わせて修正する。妥協しない

## Purpose

- three.js で PS2 起動画面のタワーシーンを完全再現する
- 動画フレームと比較可能な精度で、構図・動き・質感を再現する
- 後続で shader / particle / postprocess を追加しやすい構成にする

## Scope

再現対象は動画の `0.0s – 9.5s` 区間に限定する。以下の要素を含む。

### 1. Prism Field（白い四角柱群）

- 白～ライトグレーの四角柱を **50–80 本** 密集配置する
- 配置はグリッド状だが、一部に欠けや間隔の揺れがある
- 高さのバリエーションが大きい（低い柱と高い柱が混在、都市のスカイラインのような印象）
- 柱同士の間隔は狭く、上から見ると面が敷き詰められたように見える
- 質感はマットな白～クールグレー。金属光沢ではなくコンクリート的

### 2. Ground Plane（地面）

- 柱群の下に暗いフラットな地面がある
- 柱の根元が見える程度の暗さ
- 反射はほぼなし、ただし柱の影を受ける

### 3. Central Glow（中央発光）

- 柱群の **中心部** から青紫の発光が放出されている
- 遠景のモヤではなく、柱の隙間から漏れ出す光源として見える
- 光はカメラが近づくにつれ存在感を増す
- この中央発光がシーン全体の神秘的な印象の核

### 4. Camera（カメラ）

- 開始: ほぼ真上（仰角 70–80°）から全体を俯瞰
- 回転: 反時計回りにゆっくり軌道回転（Y 軸周り）
- ズーム: 徐々に柱群に近づいていく
- `~7.5s` から回転・ズームが急加速
- `~9.0s` で柱に非常に近い位置まで接近し、暗転に入る

### 5. Floating Cubes（浮遊キューブ）

- 柱群の間に **3–5 個** の半透明キューブが浮遊
- 色は暗い半透明（ダークグレー / ダークブルー寄り、ガラスのようなクリアさではない）
- ゆっくり回転しながら漂う
- 柱の上ではなく柱群の **間** に浮いている

### 6. Particles（パーティクル光線）

- 赤・緑・青紫の細い **ライン状の軌跡**（点ではなくレーザー的な線）
- 柱群の間を水平方向に横切るように移動する
- 数は少なく（3–6 本程度）、静かな存在感
- 残像として尾を引く表現

### 7. Ending（暗転）

- `~7.5s` から加速開始
- `~8.5s` からシーン全体が暗くなり始める
- `~9.5s` で完全暗転（純黒）

## Non-Goals

- "Sony Computer Entertainment" テキスト表示
- "PlayStation 2" ロゴ表示
- 音声・BGM の再現
- 写実的な背景（動画の背景は純黒）
- 過剰な bloom や派手なポストエフェクト
- 完全な物理シミュレーション

## Technical Requirements

- Three.js（React Three Fiber）で実装
- リアルタイム描画で安定動作
- 外部 3D モデルは使用しない（procedural geometry のみ）
- resize 対応
- コードは後から各パラメータを微調整しやすい構成

## Deliverables

- タワーシーン完全再現（0.0s – 9.5s 区間）
- アニメーション制御（カメラ軌道 + 加速 + 暗転）
- 中央発光、パーティクル光線、浮遊キューブを含む演出

## Success Criteria

- **動画の任意のフレーム** とスクリーンショットを並べて、同一シーンと認識できる
- カメラ軌道（開始角度→軌道回転→加速→接近）が動画と一致する
- 中央の青紫発光が柱の隙間から漏れ出す形で表現されている
- 暗転のタイミングと速度が動画と一致する
- mid-range ラップトップ GPU で安定 60fps
- 検証フロー（`docs/VERIFICATION.md`）のすべてのチェックポイントを通過する

## Reference Frames

実装時に以下のタイミングで動画フレームとの比較を行う。

| Time | Description | Key Check |
|------|-------------|-----------|
| 0.0s | 初期フレーム | 俯瞰角度、柱配置の密度 |
| 1.0s | 回転初期 | 柱の見え方の角度、中央発光 |
| 4.0s | 中盤 | パーティクル軌跡、浮遊キューブの位置 |
| 7.0s | 加速直前 | ズーム具合、柱のサイズ感 |
| 8.5s | 加速中 | ブラー感、暗転の進行度 |
| 9.5s | 暗転完了 | 純黒 |
