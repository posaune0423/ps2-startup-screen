# PRD

## Product Name
PS2-style Startup Scene (Three.js)

## Goal
有名なPlayStation 2の起動画面をモチーフにした、抽象的でミステリアスな3Dシーンを three.js で高精度に再現する。

## Purpose
- three.js で再利用可能な起動画面風シーンを作る
- 構図・動き・質感を整理し、実装ブレを防ぐ
- 後続で shader / particle / postprocess を追加しやすくする

## Scope
実装対象は以下に限定する。

- 白い四角柱群
- 上方から見下ろすカメラ
- カメラ回転とズーム
- 背景の青紫系モヤ
- 浮遊する透明立方体
- 発光粒子と残像
- 終盤の加速と暗転フェードアウト

## Visual Requirements

### 1. Prism Field
- 白い四角柱を約30本配置する
- 配置は碁盤状に近い規則的レイアウト
- 各柱の高さはランダムで異なる
- 質感は白、ややソフト、金属感は強くしない

### 2. Camera
- カメラは斜め上から全体を俯瞰する
- 反時計回りにゆっくり回転する
- 同時にゆっくりズームインする
- 8秒付近から回転とズームが急加速する

### 3. Atmosphere
- 背景に青紫〜群青色のモヤを配置する
- モヤは発光感のある煙・雲のように見せる
- 背景全体は暗めで、奥行きを感じる色調にする

### 4. Floating Cubes
- 一部の柱の上に透明立方体を2〜3個浮かせる
- ガラスやアクリルのようなクリア感を持たせる
- わずかに漂うような動きを入れる

### 5. Particles
- 赤、群青、緑の発光粒子を飛ばす
- 粒子は自由に漂うように移動する
- 残像や尾を引く表現を入れる
- 派手すぎず、静かな存在感にする

### 6. Ending
- 8秒目あたりから全体演出を加速する
- 同時に画面を暗くしていく
- 最後は自然にフェードアウトする

## Technical Requirements
- Three.js を使用する
- リアルタイム描画で動作する
- 外部3Dモデルは使用しない
- 基本は procedural に構築する
- コードは後から調整しやすい構成にする
- resize 対応を入れる

## Non-Goals
- UI や文字表示
- ロゴ表示
- 写実的な背景
- 過剰な bloom や派手な演出
- 完全な物理シミュレーション

## Deliverables
- three.js 実装コード
- シーン構成
- アニメーション制御
- モヤ、粒子、フェードを含む最低限の演出

## Success Criteria
- 一目であのPS2風の起動画面を想起できる
- 構図、動き、暗転の流れが自然
- 柱、モヤ、粒子が破綻せず同居する
- 実装が後続の調整に耐えられる
